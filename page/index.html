• Deployment of a web application on the cloud using CI/CD Pipelining

Abstract

In general, the developed projects or applications are deployed on servers maintained in offices. This approach has a huge drawback as the servers need to be maintained with utmost care and when there is a huge traffic of hits servers slow down. To speed up, the servers need to be upgraded or need to be increased. Additionally, servers are consuming a lot of energy, which is leading to an increase in cost. The main reason for this is many companies are still using old servers by providing them with additional cooling equipment instead of moving to new servers or technology. The database used by these applications like MS SQL Server, and Oracle also need to be maintained in the servers. The deployment process also needs to be done manually, which requires additional effort. To overcome these issues, the cloud platform is the best solution for maintaining the database and deployment. The deployment can also be automated by using Jenkins, which is an open-source platform that helps in the automation of continuous integration and continuous delivery of applications.

Technologies in DevOps

CI/CD Pipelining

A continuous integration and continuous deployment (CI/CD) pipeline is a series of steps that must be performed in order to deliver a new version of software. CI/CD pipelines are a practice focused on improving software delivery throughout the software development life cycle via automation.

Jenkins

Jenkins is a self-contained, open source automation server which can be used to automate all sorts of tasks related to building, testing, and delivering or deploying software. Jenkins can be installed through native system packages, Docker, or even run standalone by any machine with a Java Runtime Environment (JRE) installed.

https://www.jenkins.io/doc/
Maven

Apache Maven is a software project management and comprehension tool. Based on the concept of a project object model (POM), Maven can manage a project's build, reporting and documentation from a central piece of information.

Maven's primary goal is to allow a developer to comprehend the complete state of a development effort in the shortest period of time. In order to attain this goal, Maven deals with several areas of concern:

  • Making the build process easy
  • Providing a uniform build system
  • Providing quality project information
  • Encouraging better development practices
https://maven.apache.org/what-is-maven.html
Apache Tomcat

Apache Tomcat is a free and open-source implementation of the Jakarta Servlet, Jakarta Expression Language, and WebSocket technologies. It provides a "pure Java" HTTP web server environment in which Java code can also run. Thus it is a Java web application server, although not a full JEE application server.

https://tomcat.apache.org/
AWS EC2

Amazon Elastic Compute Cloud is a part of Amazon.com's cloud-computing platform, Amazon Web Services, that allows users to rent virtual computers on which to run their own computer applications.

Github

GitHub, Inc. is an Internet hosting service for software development and version control using Git. It provides the distributed version control of Git plus access control, bug tracking, software feature requests, task management, continuous integration, and wikis for every project.
